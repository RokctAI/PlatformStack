# ROKCT Platform Ecosystem Repositories

This document serves as the absolute Source of Truth for all active repositories required to build, test, provision, and deploy the decentralized **ROKCT Platform** (comprising the Control Plane, Tenant Spokes, IoT Edge modules, and the ROK agent ecosystem).

---

## 1. Core Platform Repositories

### 1.1 [PlatformStack](https://github.com/RokctAI/PlatformStack.git)
*   **Active Branch**: `main`
*   **Role**: Docker orchestrations, base images baking (`rpanel-control`, `rpanel-tenant`, `drone-brain`, `rpanel-db`), global database migrations, and the authoritative `build_ecosystem.sh` pipeline.
*   **Key Path**: [platform/Dockerfile](file:///C:/Users/sinya/Desktop/RokctAI/PlatformStack/platform/Dockerfile)

### 1.2 [Control Plane App (`control`)](https://github.com/RokctAI/control.git)
*   **Active Branch**: `main`
*   **Role**: The primary administrative hub site (`platform.rokct.ai`). Manages tenant container provisioning, collection/subscription lifecycles, hosting the Host-Level WhatsApp Bridge, and single-use secure secrets boot handshakes.

### 1.3 [Tenant Spoke App (`rcore`)](https://github.com/RokctAI/rcore.git)
*   **Active Branch**: `main`
*   **Role**: The ERP core backend for tenant spoke instances (`*.tenants.rokct.ai`). Houses the dynamic `StartupOS` Strategic Onboarding compiler, comparative telemetry engines, and in-memory bootstrap secrets managers.

### 1.4 [Next.js Web Frontend](https://github.com/RokctAI/RokctAI_frontend.git)
*   **Active Branch**: `main`
*   **Role**: The premium, glassmorphic Next.js portal. Features type-safe server actions dynamically generated by `fetch-schemas.mjs`, real-time Server-Sent Events (SSE) WhatsApp pairing streams, and stateless context rolling.

### 1.5 [IoT Edge Spoke (`Monorepo`)](https://github.com/RokctAI/Monorepo.git)
*   **Active Branch**: `main`
*   **Role**: Coordinates lightweight Go-based edge spokes (`drone-edge`) and compliance profiles for IoT POS deployments. Also serves as our local development whitelist and blueprint ledger.

### 1.6 [The ROKCT Protocol Submodule](https://github.com/RokctAI/The-Rokct-Protocol.git)
*   **Active Branch**: `main`
*   **Role**: Houses the core immutable templates, dynamic gender/pronoun mappings, and the authoritative `compiler.py` and `parser.py` modules.

## 2. ROK Agent & Adapter Repositories

### 2.1 [ROK Agent Core](https://github.com/RokctAI/ROK.git)
*   **Active Branch**: `rokct` (Customized Fork)
*   **Role**: The primary AI agentic coding and completions engine. Bundled with the host-level WhatsApp bridge and dynamic terminal executors. Kept in sync with upstream `NousResearch/hermes-agent`.

### 2.2 [ROK Paperclip Adapter](https://github.com/RokctAI/rok-paperclip-adapter.git)
*   **Active Branch**: `rokct` (Customized Fork)
*   **Role**: A lightweight Node.js/TypeScript adapter implementing standard multi-agent interfaces. Allows the centralized ROK agent to act as a managed employee inside a Paperclip company.
*   **Upstream Reference**: [NousResearch/hermes-paperclip-adapter](https://github.com/NousResearch/hermes-paperclip-adapter.git)

### 2.3 [Paperclip Core (Upstream Core)](https://github.com/NousResearch/paperclip.git)
*   **Active Branch**: `main` (or target release)
*   **Role**: The central multi-agent chat interface and coordination server. **This is the actual Paperclip host application.** ROK-paperclip-adapter hooks ROK directly into this environment to let it receive tasks and execute tickets.
